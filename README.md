# CS320-Software-Test-Automation
Java Contact Service project with JUnit testing and software testing reflection demonstrating secure design, requirement analysis, and quality assurance practices.
Project 1: contact service

Contact.java, ContactService.java, ContactTest.java, ContactServiceTest.java

Project 2:&nbsp;

These artifacts demonstrate my ability to design and implement object-oriented Java classes, apply input validation and enforce requirements, develop unit tests using JUnit, apply software testing strategies, analyze and improve software quality.

To ensure functionality, I rely on systematic testing, especially unit testing. In Project One, I created JUnit test cases to validate both normal behavior and edge cases. Writing automated tests helped uncover potential defects early and ensured that all requirements were met.&nbsp;To ensure security and integrity, I implemented input validation within constructors and service methods. For example, enforcing constraints such as maximum field lengths and preventing null values protects the system from invalid or potentially harmful data. I also followed encapsulation principles by keeping fields private and controlling access through methods. Combining defensive programming techniques with automated testing improves both reliability and security.

I begin by carefully reviewing the requirements and translating them into functional behaviors. For the Contact Service project, I analyzed constraints such as character limits, required fields, and unique contact IDs. I then mapped these requirements directly into validation logic and test cases.&nbsp;I also use test cases to represent user expectations. By writing tests that reflect user needs, I ensure the program behaves as intended. This approach helps bridge the gap between written requirements and working software.

I approach software design by breaking the system into smaller, manageable components. In this course, I separated the Contact class (data model) from the ContactService class (business logic). This separation of concerns improves maintainability and scalability.&nbsp;I also design with testing in mind. Creating test cases alongside development helps ensure each component works independently before integration. I prioritize clarity, modularity, and adherence to requirements. By structuring code logically and validating inputs early, I create systems that are easier to maintain, debug, and extend.
